**OpenAI** ประกาศ (8 ก.ค.) เปิดตัว **GPT-Live** เจเนอเรชันใหม่ของ voice model ที่ทำให้บทสนทนากับ ChatGPT ใกล้เคียงกับคุยคนจริงมากขึ้น. สถาปัตยกรรมเป็น **full-duplex** — โมเดล **ฟังและพูดพร้อมกันได้** ไม่ใช่สลับกันแบบวอล์กี้ทอล์กี้; ระหว่างสนทนา GPT-Live ใช้ backchannel เช่น "mhmm", "yeah" เพื่อบอกว่า "ตั้งใจฟัง", สามารถโต้ตอบเร็ว ๆ ได้ หรืออยู่นิ่งเมื่อผู้ใช้กำลังคิด. Full-duplex เป็นจุดเปลี่ยนของ voice UX ปี 2026 — เปลี่ยนจาก "walkie-talkie" ที่ต้องพูดจบแล้วรอ endpoint เป็นบทสนทนาที่มี social signal จริง; ในห้องเรียน HCI นี่คือเคสสอนว่าทำไม backchanneling ทำให้ agent รู้สึกว่า "ตั้งใจฟัง" มากกว่า ASR pipeline classic. เชิงสถาปัตยกรรม สิ่งที่เปลี่ยนคือแยก encoder เสียงเข้ากับ decoder เสียงออกให้ทำงาน parallel + policy ตัดสินใจว่าเมื่อไหร่ควรตอบสนอง เมื่อไหร่ควรเงียบ — ต่างจาก ASR→LLM→TTS pipeline ที่ latency เป็น serial sum, จุดชี้ขาดที่ต้องจับตาคือ end-to-end turn latency และ interruption handling. สำหรับทีมที่ integrate voice agent อยู่ ต้อง redesign UX จาก push-to-talk เป็น streaming — API contract เปลี่ยน; ต้อง handle bi-directional audio stream + event stream (agent เริ่มพูด, ผู้ใช้ interrupt) พร้อมกัน, ทดสอบ echo cancellation และ VAD บน browser/mobile ก่อนคิดจะ ship
