### 2. MCP 2026-07-28 spec ออก release candidate — stateless core + MCP Apps / Tasks — [TechCrunch](https://techcrunch.com/2026/07/20/ais-most-important-protocol-is-getting-a-little-bit-easier-to-use/)

TechCrunch รายงานว่า **Model Context Protocol (MCP)** — โปรโตคอลที่ AI model ใช้เข้าถึงข้อมูลและเครื่องมือภายนอก — กำลังออก **release candidate ของ spec วันที่ 2026-07-28** ซึ่งเป็นการปรับใหญ่ที่สุดตั้งแต่เปิดตัว. หัวใจของ update คือ **session ID handling แบบใหม่** ที่ให้ server รันแบบ stateless ได้จริง: remote MCP server ที่เดิมต้องพึ่ง sticky session, shared session store, และ deep packet inspection ที่ gateway ตอนนี้รันหลัง **plain round-robin load balancer** ได้ทันที — routing ทำผ่าน `Mcp-Method` header และ client cache `tools/list` ตาม `ttlMs` ที่ server ประกาศ. Spec ยังเพิ่ม extensions สำหรับ server-rendered UI (**MCP Apps**), long-running work (**Tasks**), และปรับ authorization ให้อิงกับ **OAuth / OpenID Connect** พร้อม formal deprecation policy.

เชิงการสอน นี่คือเคส distributed-systems 101 ที่จับต้องได้ — จาก session-affinity ที่ตัดขวาง horizontal scale ไปสู่ stateless HTTP + client-side caching; MCP กำลังทำสิ่งที่ REST เคยทำในยุค 2000s ให้ agent tooling, ใช้เป็นเคสสอน API design และ protocol evolution ได้ตรง. ประเด็นเชิง AI ที่ลึกกว่า header ก็คือ MCP เริ่มยอมรับตัวเองว่าเป็น **standard ระดับ production**: การมี OAuth/OIDC alignment, formal deprecation policy, และ extensions ที่ครอบคลุม UI + long-running task คือการปิด gap ให้ enterprise รับ MCP ไปวาง production ได้จริง หลังจาก 12 เดือนที่ยังอยู่ในสาย OSS experimental. สำหรับโปรแกรมเมอร์มืออาชีพที่รัน MCP server อยู่ ก่อน 28 ก.ค. ควรวาง migration plan: เปลี่ยน infrastructure จาก sticky-session gateway → plain load balancer + `Mcp-Method` routing, เพิ่ม `ttlMs` ให้ `tools/list`, และทดสอบ auth flow ใหม่ที่ผูกกับ OAuth resource server ให้ครบก่อนย้าย traffic — ประหยัด infra cost + ลด latency ทันทีเมื่อย้ายเสร็จ.
