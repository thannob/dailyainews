### 1. Alibaba สั่งห้ามพนักงานใช้ Claude Code เริ่ม 10 ก.ค. — สัญญาณแยก dev-tool stack ระหว่างจีนกับสหรัฐฯ — [TechCrunch](https://techcrunch.com/2026/07/04/alibaba-reportedly-bans-employees-from-using-claude-code/)

รายงานเมื่อ 4 ก.ค. 2026 ระบุว่า **Alibaba** จะห้ามพนักงานใช้ **Claude Code** เครื่องมือ agentic coding ของ Anthropic ตั้งแต่ 10 ก.ค. เป็นต้นไป พร้อมสั่งลบโมเดล Claude ออกจากคอมพิวเตอร์บริษัท ตามรายงานอ้างว่านักวิจัยด้านความปลอดภัยเจอ code path ในเครื่องมือของ Anthropic ที่แยกผู้ใช้ในจีนออก (ผ่าน timezone และโดเมน `.cn`) ซึ่งบริษัทจีนตีความว่าเป็นความเสี่ยงข้อมูลรั่วสู่รัฐบาลสหรัฐฯ; Anthropic เองก็ห้ามบริษัทจีน (รวมทั้งบริษัทลูกในต่างประเทศที่จีนเป็นเจ้าของ) ใช้โมเดลของตนอยู่แล้ว. Alibaba ผลักดันให้พนักงานใช้ **Qoder** และเครื่องมือสาย **Qwen-Coder** ของตนเองแทน.

เคสนี้เป็นตำราสั้น ๆ ของ "geopolitics of tooling" — developer tool ไม่ใช่แค่เครื่องมือ แต่กลายเป็น **attack surface** ที่รัฐและ enterprise มองต่างกัน อย่างที่เคยเกิดกับ Huawei, ARM, และการ block GitHub บางบัญชี รอบนี้ไปถึงระดับ **โค้ดที่โปรแกรมเมอร์พิมพ์ทุกวัน**. เชิงเทคนิคประเด็นจริงคือ **telemetry ที่ฝังใน agentic coding tool** — Claude Code เรียก tool call กับ filesystem, network, และ shell; หาก logic แยก user by locale มีจริงก็ควรถูก audit อย่างเป็นระบบ (ไม่ใช่เชื่อคำอ้างเดียว) เพราะห่างจากกันไม่มากระหว่าง **locale routing ตามปกติ** กับ **backdoor**. สำหรับทีมที่มี dev ใน CN region หรือ enterprise ที่จริงจังเรื่อง data residency: ก่อน adopt agentic CLI ตัวไหน ให้ **network-egress audit** และ **on-device static scan** เป็น check-list มาตรฐาน; ทำ fallback plan (Qwen Coder, Copilot on-prem, local Aider + open-weight model) ที่ทีมสามารถสลับได้ภายใน 1 sprint — เพราะ tooling risk รอบนี้เกิดขึ้นเป็น **event ในระดับสัปดาห์** ไม่ใช่ **แนวโน้มในระดับปี**.

### 4. Micron วางเสาเข็มโรงงาน HBM $9.3B ที่ฮิโรชิม่า — สัญญาณ HBM shortage ยาว 2+ ปี — [Bloomberg](https://www.bloomberg.com/news/articles/2026-07-04/micron-breaks-ground-on-9-billion-western-japan-plant-expansion)

โฟกัสของสื่อไปที่ compute (GPU/TPU) แต่ **bottleneck จริงหลายครั้งคือ HBM** — เพราะโมเดล frontier ตัดสินด้วย memory bandwidth, ไม่ใช่ raw FLOPS อีกต่อไป. เชิง supply chain ที่ ship ปี 2028 แปลว่าอุตสาหกรรมมองว่า **HBM shortage ยาวอย่างน้อย 2 ปี**; ถ้าเป็นจริง inference cost ของ frontier model จะยัง sticky, และการ optimize **model architecture (MoE, quantization, KV cache, speculative decoding)** ยังเป็นงาน first-order — คนที่วัด memory bandwidth ต่อดอลลาร์เป็น metric สำคัญกว่า FLOPS ต่อดอลลาร์จะได้เปรียบใน 24 เดือนข้างหน้า. เชิง geopolitics การเลือก **ฮิโรชิม่า** ไม่ใช่ Arizona หรือ Idaho บอก signal ว่า US CHIPS Act ยังไม่ครอบคลุมทุก node — ญี่ปุ่นยังเป็น **anchor memory** ในโครงข่ายที่สหรัฐฯ ยอมพึ่ง. สำหรับทีม engineering: cloud AI price ยังไม่ collapse ในเร็ววัน; **capacity planning** สำหรับ product ที่พึ่ง frontier model ควรมี scenario "supply-constrained region" ที่ต้อง reroute traffic ข้าม region หรือ fallback ไปโมเดลเล็กกว่าไว้ใน architecture ตั้งแต่ตอนนี้ — ไม่ต้องรอเจอ throttle จริงถึงเริ่มออกแบบ.
