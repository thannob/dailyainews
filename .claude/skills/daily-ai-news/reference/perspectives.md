# Perspectives — 2026-07-03

## 1. Microsoft Frontier Company — $2.5B, 6,000 forward-deployed engineers

**อาจารย์ (มหาวิทยาลัย):** เคสสอนแนว "organizational form follows technology" ที่ชัดเจนที่สุดของครึ่งปีหลัง — เมื่อ product เดิม (Copilot, Azure AI) ยังไม่แปลงเป็น revenue จริงในลูกค้าเพราะ deployment gap, Microsoft ตอบด้วยการตั้ง entity แยกและใช้ forward-deployed engineer model แบบ Palantir ให้เห็น. เอาไปคู่กับ Chandler "Structure and Strategy" ให้นักเรียนวิเคราะห์ว่ารอบนี้ structure เปลี่ยนไล่ตาม strategy กี่เดือน.
**ผู้เชี่ยวชาญด้าน AI:** สัญญาณสำคัญคือ **hyperscaler ยอมรับว่า SaaS distribution ไม่พอ** สำหรับ enterprise AI — ต้องมี human-in-the-loop ระดับ 6,000 คนถึงจะทำ integration ให้สำเร็จ ซึ่ง imply ว่า model capability ล่าสุดยัง gap เยอะจนต้องใช้วิศวกร wrap around. Amazon ทำ $1B FDE unit สองวันก่อน = เป็น pattern industry, ไม่ใช่ตัวเลือกของ Microsoft คนเดียว.
**โปรแกรมเมอร์มืออาชีพ:** ถ้า customer ของคุณเป็น enterprise ที่ใช้ Microsoft stack — คาด Frontier engineer จะเดินเข้ามาชิงงาน integration ที่คุณเคยรับ contract ทำ. Compete ด้วยการ specialize domain-specific evaluation/agent architecture ที่ Microsoft-generic engineer ทำไม่ได้ — หรือหาทางเป็น partner ใน ecosystem ก่อน 6,000 คนเต็ม.

## 2. Anthropic × Samsung Foundry — คุยผลิตชิป custom 2nm

**อาจารย์ (มหาวิทยาลัย):** ในห้อง Strategy เอาเคสนี้ต่อจาก Andrew Grove "vertical integration is a strategic choice" — Anthropic ที่ recently ปิดรอบทุน $65B และมี investor Samsung/SK hynix/Micron อยู่ในรอบ กำลัง signal ว่า **ownership of the silicon layer คือ moat ระดับใหม่ของ frontier lab** ไม่ใช่แค่ model weight. อ่านคู่กับ OpenAI × Broadcom Jalapeño (มิ.ย. 2026) แล้วให้เขียนว่าใครมี structural advantage มากกว่า.
**ผู้เชี่ยวชาญด้าน AI:** technically ยัง early — Anthropic ยัง**ไม่ตัดสินใจ**ด้วยซ้ำว่าชิปจะทำ training หรือ inference. แต่ signal ที่มี weight คือ Samsung 2nm node = leading edge ที่ TSMC ครองอยู่, การเลือก Samsung = ทั้ง alternative supply chain + geopolitical hedge (Korea vs Taiwan). ระวังอย่าอ่าน timeline ผิด — จาก concept ถึง production ~24-30 เดือน; ผลกระทบต่อ Nvidia dependency ยังไม่ถึง 2028.
**โปรแกรมเมอร์มืออาชีพ:** ผลระยะสั้นต่อ Claude API pricing = ศูนย์. ผลระยะกลาง (2027-28) = Anthropic น่าจะมี pricing tier ที่ optimize สำหรับ workload ที่ chip ตัวใหม่ทำได้ดี — ถ้าออกแบบ product ที่ inference cost คือ COGS หลัก ให้ตาม roadmap ชิปของ vendor ทุกเจ้า (OpenAI Jalapeño, Anthropic-Samsung, Google TPU, AWS Trainium/Inferentia) เพราะ pricing curve จะแยกตาม silicon ที่รัน.

## 3. Google emissions +25%, Amazon +16% — AI จริงจนสมุด Environmental Report บอก

**อาจารย์ (มหาวิทยาลัย):** เคสที่ต่อโดยตรงเข้าห้อง Corporate Sustainability / ESG — 2 บริษัทที่ประกาศ net-zero (Google 2030, Amazon 2040) เพิ่ม emission ในทิศทางที่**ตรงข้ามกับเป้าหมาย** เพราะ AI. ให้นักเรียนอ่าน Google's 2026 Environmental Report ตรง (blog.google) แล้วถามว่า "materiality" ของ target ยังคงเดิมได้อยู่หรือไม่ในเงื่อนไข AI demand growth — ประเด็นที่ auditor และ ESG rating agency จะต้องคิดใหม่ทั้งหมด.
**ผู้เชี่ยวชาญด้าน AI:** ตัวเลข +25% / +16% ในหนึ่งปีเป็นเรื่องใหญ่ทาง trend — และเรื่องนี้เชื่อมกลับข่าว Nvidia × Valar (nuclear-powered compute) ของเมื่อ 1 ก.ค.: hyperscaler กำลังหา behind-the-meter power เพราะ grid + PPA แบบเดิมไม่พออีกต่อไป. คำถามที่ยังไม่มีคำตอบ — ถ้า inference-per-token efficiency ปรับปรุงเร็ว (โมเดลเล็กลง, distillation ดีขึ้น) จะกินการเติบโตของ demand ทันหรือไม่.
**โปรแกรมเมอร์มืออาชีพ:** ถ้า ESG requirement ของบริษัทคุณเริ่ม audit ทาง supply chain — เตรียมข้อมูล **carbon-per-1M-token** ของ provider ที่คุณใช้ให้พร้อม; Google Cloud, AWS, Azure จะเริ่มให้ตัวเลขนี้ผ่าน SDK / billing metadata ภายใน 12 เดือน. Product ที่ประหยัด token (prompt caching, response streaming, model routing to smaller tier) จะกลายเป็นทั้ง cost play และ ESG play — พูดออกไปในทีมได้แล้วในสัปดาห์นี้.

## 4. Meta Pocket — vibe-coded gaming/app จาก prompt

**อาจารย์ (มหาวิทยาลัย):** เคสสั้นๆ ที่สอนได้ทั้ง (1) M&A → productization: Meta ซื้อ Gizmo (vibe-coded gaming platform) ต้นปี → ออก Pocket ครึ่งปี; (2) product strategy ของ Meta ที่ยังพยายามหา **consumer AI hit** หลัง Meta AI ในแอปเดิมยังไม่แจ้งเกิดเต็ม. ให้นักเรียนเทียบกับกลยุทธ์ Google (Gemini into everything) และ Apple (Apple Intelligence, on-device) ว่าใคร framing ผู้ใช้ทั่วไปได้ประสบผลจริง.
**ผู้เชี่ยวชาญด้าน AI:** vibe-coding = pattern ที่ผู้ใช้ทั่วไป**ไม่ต้องเขียนโค้ด**แต่ได้ artifact ที่ interactive; นี่ไม่ใช่ innovation ระดับ model แต่เป็น product-form innovation ที่ใช้ประโยชน์จาก Llama + tool-use ที่ Meta มี. คำถามธุรกิจที่แหลม — Meta จะ monetize ยังไง (ads inside generated games?), และ generated content moderation จะจัดการอย่างไรในสเกล user 3 พันล้านคน.
**โปรแกรมเมอร์มืออาชีพ:** สัญญาณอ่อนที่ควรจับ — vibe-coded platform ระดับ mass consumer (Pocket, Google Gemini Canvas, Anthropic Claude Artifacts) เริ่มบีบให้ **long tail ของ hobby project developer หายไปเป็นสองทาง**: (1) ลดจำนวนคนที่จะเริ่มเรียน code เพราะ prompt พอสำหรับสิ่งง่ายๆ, (2) ยกระดับ threshold ของ "จริงจัง" ที่เดฟที่เป็น professional ต้องอยู่ให้สูงกว่าที่ vibe-coding ทำได้. Position ตัวเองในสองทาง — เข้าใจ system, deploy, scaling, security — ซึ่ง Pocket ไม่ทำ.
